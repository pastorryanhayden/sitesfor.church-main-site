How it works

We don't make TRADITIONAL websites.

We make STATIC websites.

On a traditional website (wordpress, joomla, wix, etc.) a program runs on your server which generates pages as they are requested. On our websites, the only thing on the server are the static files that make up your files and they are regenerated every 30 minutes.

“But that doesn’t sound like a feature…”

Actually, it is. Static sites have a ton of advantages including:

  1. Speed:

    Your site will run faster and serve faster to potential visitors.

  2. Security:

    Because your site is just a bunch of static files - it’s almost impossible to hack.

  3. Simplicity:

    There are far less moving parts to break and when something does break - it doesn’t effect your live site. Also, because it's just HTML files, it's far easier to customize.

  4. Savings:

    Static sites use less resources and are thus cheaper to run.

We aren’t the only people who are noticing this. Static websites have taken off over the last two years and are being used by everyone from the president to fortune 500 companies.

We use Airtable

Traditional websites run a Content Management System (CMS) like Wordpress, Joomla, Concrete5, etc. The CMS is special software that allows you to update your website.

We use Airtable - a cloud based database program - instead of a content management system. Our computers look for changes to your data, and then rebuilds your site with that new data and pushes it to a webhost.

“But that doesn’t sound like a feature…”

Airtable is a fantastically easy to use online database program. If you’ve ever used a spreadsheet - you’ll understand Airtable right away. Unlike a spreadsheet, however, Airtable has some killer features like:

  • Drag and Drop file uploading.
  • uilt in basic image editing.
  • Special fields like checkboxes and menus.
  • Database relationships.
  • History rollbacks.
  • Fantastic mobile apps for iOS and Andriod.
  • Very robust sharing features.

Most Content Management Systems are clunky, insecure and have way too many confusing features. We think Airtable is a better way to manage your site.

We use MARKDOWN

Traditional websites use some kind of WYSIWYG (What you see is what you get) form for styling content.

We use MARKDOWN, an easy to learn and far easier to use plain text writing syntax to acheive the same thing.

“Syntax??? That sounds terrifying!”

Most website providers provide a way for you to add text and ours is no different. What’s different about Sitesfor.church way is rather than using an interface we use a syntax. It’s very simple and easy to learn.

You can write Markdown directly in Airtable or in any text-editor, but we recommend using a web app like Classeur.io a Mac app like Byword or a windows app like Typora.
